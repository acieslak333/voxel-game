# code-index

Keep this repository navigable and self-documenting. The index has **three
layers** (see `docs/CODE_INDEX_GUIDE.md` for the full rationale):

- **A. Machine index** — `docs/index/*`, generated by `tools/codeindex/codeindex.py`.
- **B. Curated prose** — `docs/CODE_INDEX.md` (hub) + `src/*/README.md`.
- **C. In-source Doxygen** — `@file` banners + per-symbol comments in `.h`/`.cpp`.

Pick the workflow that matches the request.

---

## Workflow 1 — Refresh the index (most common)

After source changed, bring every layer back in sync.

```bash
python tools/codeindex/codeindex.py gen      # regenerate docs/index/*
python tools/codeindex/codeindex.py check    # confirm clean (exit 0)
```

Then review whether the change altered a subsystem's **shape** (new/moved/deleted
file, new cross-cutting concern). If so, update:
- the relevant table in `docs/CODE_INDEX.md`, and
- the matching `src/<dir>/README.md`.

Commit `docs/index/` together with the source change. Done.

---

## Workflow 2 — Check for staleness / drift

```bash
python tools/codeindex/codeindex.py check    # exit 1 + drift list if stale
```

- If it reports added/removed/changed files → run Workflow 1.
- Also scan for **doc drift** (docs describing code that no longer exists). Grep
  the claim against `src/`; if the code disagrees, the **code wins** — fix the doc
  or add it to the [Documentation drift](../../docs/CODE_INDEX.md#documentation-drift)
  section of the index. (Known examples: `TerrainGenerator`, `biomes.yaml`,
  `--genmap`, deleted worldgen tools.)

---

## Workflow 3 — Add Doxygen comments to source (Layer C)

This is a **documentation-only** change. Read §4 of `docs/CODE_INDEX_GUIDE.md`
for the template and rules. The non-negotiables:

1. Comment-only — never change/reorder/delete a line of code. (You may convert an
   existing `//` top banner into an `@file` block.)
2. No trailing comments on existing code lines — member docs go on a `///` line
   *above* the member.
3. Never break a preprocessor line-continuation (`\`); skip vendored files
   (`utilities/noise/FastNoise.*`, `third_party/`) beyond a one-line `@file` note.
4. Accuracy over volume; `@warning`/`@note` for threading/determinism invariants.

For a **large pass**, partition `src/` by subsystem and delegate disjoint file
lists to parallel `cpp-pro` sub-agents (so no two touch the same file), each given
the template + rules. Skeleton prompt: see `docs/CODE_INDEX_GUIDE.md` §4.3.

**Always verify before committing** (the build is Windows/VS-only, so this is how
a doc pass is proven safe without a compiler):

```bash
python tools/codeindex/verify_comments.py    # strips comments, asserts code is byte-identical vs HEAD
```

If it prints `CODE CHANGED` for a file, that file has a real edit — fix it.

---

## Workflow 4 — Render the Doxygen HTML

Requires `doxygen` installed (not needed to build the game):

```bash
cmake --build build --target docs     # or: doxygen Doxyfile
# → docs/doxygen/html/index.html
```

---

## Adding a file — checklist

1. `@file` banner + Doxygen on public symbols.
2. `codeindex.py gen` → commit `docs/index/`.
3. Add to `src/<dir>/README.md` (and `docs/CODE_INDEX.md` if the shape changed).
4. New `.cpp` → add to `CMakeLists.txt`.
5. `verify_comments.py` if existing files were touched.
